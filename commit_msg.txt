feat(core): implement TaskContext metadata injection and FileRelations auto-maintenance

This commit implements P0 phase of TaskContext feature, including:

1. TaskContext Metadata Injection (Lazy-loading strategy)
   - Fix buildParams() bug: properly populate callback parameters
   - Extend TaskAggregate.create() to accept filename, contentType, size
   - Implement populateContextForPlugins() for metadata injection
   - Update TaskService.createTask() to pass file metadata

2. FileRelations Auto-maintenance (Domain Event approach)
   - Create DerivedFilesAddedEvent domain event
   - Implement FileRelationsEventHandler to auto-create relations
   - Support dual-reference design (sourceKey + currentMainKey + derivedKeys)
   - Add existsByFileFkeyAndRelatedFkey() to FileRelationRepository
   - Detect derived files in DefaultCallbackChainRunner and publish events

3. Spring Event-based Message Publishing (Test environment)
   - Create SpringEventCallbackPublisher with @Profile("test")
   - Create CallbackTaskEventListener with @Async execution
   - Add @Profile("!test") to Kafka publishers/consumers
   - Configure async thread pool in application-test.yml

4. Test Infrastructure Updates
   - Update PluginCallbackScenarioTest to use Awaitility
   - Replace synchronous callback execution with async polling
   - Update TaskEventPublisher interface and implementations

Key architectural decisions:
- Use lazy-loading to inject metadata only when needed (completeUpload)
- Use domain events to decouple FileRelations maintenance from TaskAggregate
- Use Spring Events in test environment to avoid Kafka dependency
- Maintain DDD aggregate boundaries (Task vs File domains)
